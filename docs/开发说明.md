# 开发说明

## 1. 目录与职责

| 路径 | 内容 |
|------|------|
| `src/`、`include/` | **唯一源码真相**；学生改 `datalink*.c/h`，教师库勿动 |
| `build/` | 编译输出（`datalink.exe`、`obj/`） |
| `Lab1-Windows-VS2017/` | 仅 `datalink.sln`、教师 `Example/` 与 `reference/` |

## 2. 与指导书章节

| 章节 | 本仓库 |
|------|--------|
| 8.2 Windows | `Lab1-Windows-VS2017\datalink.sln` → 输出 `build\datalink.exe` |
| 8.3 Linux | 根目录 `make` → `build/datalink` |
| 8.4～8.14 | `include/protocol.h` 与 `src/protocol.c` |

## 3. 调试建议

- 双端命令行选项（`-f`、`-u`、`-b` 等）必须一致。
- 建议 `-d3`，日志为 `datalink-A.log` / `datalink-B.log`（在运行目录生成）。
- 长时间无利用率输出时，检查网络层是否被长期 `disable` 或对称死锁。

## 4. 教师参考程序

`Lab1-Windows-VS2017\Example\` 下有 `stopwait.exe`、`gobackn.exe`、`selective.exe` 及样例日志，用于对照行为（指导书 6、9 节）。**不要**与 `build\datalink.exe` 混淆。

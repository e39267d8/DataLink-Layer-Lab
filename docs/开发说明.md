# 开发说明

## 1. 与《计算机网络实验一》的对应关系

| 指导书章节 | 本仓库 |
|------------|--------|
| 8.1 程序总体结构 | `protocol.c` / `protocol.h` 仿真分层 |
| 8.2 Windows 编译运行 | `Lab1-Windows-VS2017\datalink.sln`，输出 `Debug\datalink.exe` |
| 8.3 Linux | 根目录 `Makefile` → `./datalink` |
| 8.4～8.14 | 库行为见 `include/protocol.h` 注释与指导书原文 |

## 2. 你们应维护的代码路径

- **仅改**：`src/datalink.c`、`include/datalink.h`（以及你们自行拆分的 `.c/.h`，若老师允许）。
- **勿改**：`protocol.c`、`lprintf.c`、`crc32.c`、`getopt.c` 等教师库（除非老师明确允许）。

## 3. 调试建议

- 双机同一套命令行选项（`-f`、`-u`、`-b`、端口等）必须一致。
- 先用 `-d3` 打开 `dbg_event` + `dbg_frame`，对照日志与指导书 8.11。
- 接收端周期性统计见指导书 8.6（`put_packet` 间隔超过约 2 秒会打一行利用率）；若长时间不再出现，优先怀疑死锁或网络层被长期 `disable`。

## 4. 参考可执行文件（理解目标用）

`Lab1-Windows-VS2017\Example\` 或 `Lab1-Windows-VS2013\Example\` 下常有教师提供的 `gobackn.exe`、`selective.exe`、`stopwait.exe` 及对应日志样例，可与你们自研程序对比行为与利用率（指导书 6、9 节）。
